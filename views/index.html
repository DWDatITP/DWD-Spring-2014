Course description

This experimental 7-week, 2 point course will provide a framework for learning how to develop and program web applications and will focus on server side development. Rather than the traditional model of in-class lectures, assignments, projects, and office hours, this course will heavily leverage existing online and offline resources (videos, articles, interactive tutorials, online courses, books, and more) with class time fully devoted to group and individual help sessions.

This course will be focused on developing and deploying server side web applications using JavaScript, Node.js with the Express framework, and NoSQL databases on cloud based infrastructure. Additional topics will include login and session management, web services and APIs, and will lightly touch on front-end web development.

Prerequisites include Comm Lab Web and Introduction to Computational Media.

Work

Each week there will be assigned reading or viewing. It’s your responsibility to learn the concepts as well as you can outside of class time. Class time will occasionally be used for light review but it is primarily intended for putting these concepts into practice, with the help of the instructor and your classmates.

Each student will work on an individual server-side application throughout the 7 weeks of the class. You should have a good idea of what sort of application you would like to work on by the second week of class.

Over the 7 weeks we’ll cover several aspects of server-side programming including:

  • Developing an express-based server
  • Serving static assets
  • Serving dynamic templates
  • Accepting user input
  • Managing state (cookies and sessions)
  • Storing information in a database (MongoDB)
  • Deploying your application to Heroku
  • Advanced topics (Cross-Origin Resource Sharing, APIs, uploading media, front-end javascript frameworks, etc.)

Your project is not required to use all the techniques that are covered in this class, but it should use the majority of them. Projects should be worked on individually. If you have a group and would like to work together, speak to me first.

Class rhythm

Every week we will share "picks" — a link, tutorial, app, stackoverflow question, or something else that was helpful for you during the week that you think someone else in class might learn from.

Every week I will cover a "programmer concept" — a small piece of advice about programming in general that you can apply to your work on your project.

Class conduct

  • This class meets for one two-hour and thirty minute session per week.
  • Assignment due dates are hard dates.
  • If you know in advance that you will not be able to attend a class, let me know beforehand.
  • Multiple unexcused absences will result in an academic warning.
  • There is a lot to learn, so come on time.

Assessment

  • Class Participation: 40%
  • Weekly Assignments: 30%
  • Final Project: 30%

Participation includes:

  • Being attentive when others are speaking during class discussions
  • Contributing to class discussions
  • Presenting in class
  • Being active on the class mailing list
  • Helping each other in class
